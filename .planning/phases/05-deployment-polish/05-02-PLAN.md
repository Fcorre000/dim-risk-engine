  <action>
Append a second service entry to render.yaml for the frontend static site. The final render.yaml must have TWO entries under `services:`:

```yaml
services:
  - type: web
    name: dim-risk-api
    env: python
    region: oregon
    plan: free
    rootDir: api
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn main:app --host 0.0.0.0 --port $PORT
    healthCheckPath: /health
    envVars:
      - key: MODEL_PATH
        value: ../models
      - key: CORS_ORIGINS
        sync: false
      - key: PYTHON_VERSION
        value: 3.10.0

  - type: web
    name: dim-risk-frontend
    env: static
    region: oregon
    plan: free
    rootDir: frontend
    buildCommand: npm install && npm run build
    staticPublishPath: dist
    routes:
      - type: rewrite
        source: /*
        destination: /index.html
    envVars:
      - key: VITE_API_URL
        sync: false
```

Key points:
- `env: static` tells Render this is a static site (served from CDN, no server process needed).
- `staticPublishPath: dist` matches the Vite build output directory.
- The `routes` rewrite rule is mandatory for React SPA — without it, direct URL navigation returns 404.
- `VITE_API_URL: sync: false` — must be set manually in Render dashboard to the live backend URL (from Plan 05-01 SUMMARY).
- `rootDir: frontend` means Render's CWD is frontend/ for build and publish.
