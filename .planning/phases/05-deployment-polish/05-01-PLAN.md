  <action>
Create render.yaml at the repo root with this exact content:

```yaml
services:
  - type: web
    name: dim-risk-api
    env: python
    region: oregon
    plan: free
    rootDir: api
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn main:app --host 0.0.0.0 --port $PORT
    healthCheckPath: /health
    envVars:
      - key: MODEL_PATH
        value: ../models
      - key: CORS_ORIGINS
        sync: false
      - key: PYTHON_VERSION
        value: 3.10.0
```

Key points:
- `rootDir: api` means Render sets CWD to api/ before build and start commands.
- `--host 0.0.0.0 --port $PORT` is mandatory — Render injects $PORT, binding to localhost silently fails health checks.
- `MODEL_PATH: ../models` points one level up from api/ to models/ (relative to the repo root on Render's disk).
- `CORS_ORIGINS: sync: false` tells Render this value must be set manually in the dashboard (it will be the frontend URL, unknown until Plan 05-02 completes).
- `healthCheckPath: /health` makes Render verify the service is up before marking the deploy successful.

Do NOT add gunicorn — uvicorn's single worker is sufficient for a free-tier stateless service; adding gunicorn complicates MODEL_PATH and app.state sharing.

  <action>
In api/main.py, replace the hardcoded `allow_origins=["*"]` with an env-var-driven list. The updated CORS middleware block must:

1. Add `import os` at the top of the imports (after existing stdlib imports).
2. Replace the `app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)` block with:

```python
_raw_origins = os.environ.get("CORS_ORIGINS", "*")
_allow_origins = [o.strip() for o in _raw_origins.split(",")] if _raw_origins != "*" else ["*"]

app.add_middleware(
    CORSMiddleware,
    allow_origins=_allow_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

This logic means:
- Local dev: `CORS_ORIGINS` not set → `_raw_origins = "*"` → `_allow_origins = ["*"]` → no change in dev behavior.
- Production: `CORS_ORIGINS = "https://dim-risk-frontend.onrender.com"` → `_allow_origins = ["https://dim-risk-frontend.onrender.com"]` → restricted to frontend origin only.
- Multi-origin support: `CORS_ORIGINS = "https://a.com,https://b.com"` → split on comma.

Also update the lifespan function to respect MODEL_PATH env var:

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    model_path_env = os.environ.get("MODEL_PATH")
    if model_path_env:
        models_dir = pathlib.Path(model_path_env)
    else:
        models_dir = pathlib.Path(__file__).parent.parent / "models"
    with open(models_dir / "xgb_classifier.pkl", "rb") as f:
        app.state.clf = pickle.load(f)
    with open(models_dir / "xgb_regressor.pkl", "rb") as f:
        app.state.reg = pickle.load(f)
    yield
```

This allows Render to set `MODEL_PATH=../models` and have it take effect without code changes.
